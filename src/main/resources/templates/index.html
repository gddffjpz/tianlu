以下是两个文件格式为.md的接口 文档的内容

1.1 产品生产数据

1.1.1 基本信息

请求路径：/viduo

请求方式：GET

接口描述：该接口用于接受生产数据

1.1.2 请求参数

格式：json

参数说明：

参数名 类型 是否必须 备注
id number 必须 生产视频ID

请求参数样例：

    {
        "id": 1
    }

1.1.3 响应数据

参数格式：application/json

参数说明：

参数名 类型 是否必须 备注
code number 必须 响应码，1 代表成功，0 代表失败
id number 必须 id
url string 必须 视频所在地址

响应数据样例：

    {
      "code": 1,
      "msg": "success",
      "data": [
        {
          "id": 33,
          "url": "232"
        }
      ]
    }

1.2 宣传数据

请求路径：/publicize

接口描述：该接口用于接受生产数据

1.2.2 请求参数

格式：json

参数名 类型 是否必须 备注
id number 必须 生产视频ID

data object[] 非必须 返回的数据
-id number 非必须 id
-url string 非必须 详细界面
-thumbnail string 非必须 展示图片

-description string 非必须 介绍

            "id": "1",

            "description": "介绍"
        },
        {
            "id": "2",
            "url": "http://jingpeizhao.cn/java/publicize/2.html",
            "thumbnail": "http://jingpeizhao.cn/java/photo/2.png",
            "title": "名字",
            "description": "介绍"
        }
      ]
    }

生产数据

1.1 生产数据

1.1.1 基本信息

请求路径：/produce

请求方式：GET

接口描述：该接口用于接受生产数据

1.1.2 请求参数

无

1.1.3 响应数据

参数格式：application/json

参数说明：

参数名 类型 是否必须 备注
code number 必须 响应码，1 代表成功，0 代表失败
data object[] 非必须 返回的数据
- start1 string 非必须 生产环节一开始时间
- start2 string 非必须 生产环节二开始时间
- start3 string 非必须 生产环节三开始时间
- end1 string 非必须 生产环节一结束时间
- end2 string 非必须 生产环节二结束时间
- end3 string 非必须 生产环节三结束时间
- abnormal number 非必须 是否有问题，1 代表无，0 代表有

响应数据样例：

    {
      "code": 1,
      "data": [
        {
          "id": 1,
          "start1": "2022-09-01T23:06:29",
          "end1": "2022-09-01T23:06:30",
          "start2": "2022-09-01T23:06:31",
          "end2": "2022-09-01T23:06:32",
          "start3": "2022-09-01T23:06:33",
          "end3": "2022-09-01T23:06:34",
          "abnormal": 0
        }
      ]
    }

1.1 报错数据

1.1.1 基本信息

请求路径：/report

请求方式：GET

接口描述：该接口用于接受生产数据

1.1.2 请求参数

无

1.1.3 响应数据

参数格式：application/json

参数说明：

参数名 类型 是否必须 备注
code number 必须 响应码，1 代表成功，0 代表失败
data object[] 非必须 返回的数据
- report1 number 非必须 生产环节一是否报错,1是有,0是无
- report2 string 非必须 生产环节二是否报错,1是有,0是无
- report3 string 非必须 生产环节三是否报错,1是有,0是无

响应数据样例：

    {
      "code": 1,
      "data": [
        {
            "id": 1,
            "report1": "0",
            "report2": "0",
            "report3": "1"
        }
      ]
    }
